What is Sinapsi


Sinapsi is a wide open platform that gives the users the ability to create and manage Macros

Macros

A Macro is something thats is composed by a Trigger and a list of Action. You can perform a lot of things with macros, an example of macro can be "if i recive a message in my smartphone, show me a notification on my PC". As you can see, in this macro there is a condition ("if i recive") to run the macro, this condition in Sinapsi is called Trigger. The Trigger is a necessary component to run the macro, in the Macro there are also a list of actions ("show me a notification").

Triggers

The Trigger is the event that cause the execution of the Macro. Triggers are device-dependent. In your smartphone you can have triggers like: "if I recive a sms", "if I recive a phone call" and so on, but this triggers cannot be present in tablet device or pc device, because theren't the physical component to recive sms and phone call.

Actions

At this point we have the event that excute the Macro, we only need to define the actions to perfom at Macro execution, and in which device to execute the actions. Actions can be somthing like: "send email to ", "Show a notification", "Tweet on my account" and so on.



Getting started with Sinapsi

What you can do with Sinapsi



Access

You can login to the Sinapsi platform or create a new account.

Create Macros

You can create new Macros, edit and delete Macros.

Disable Macros

You can tempoparily disable Macros.

Synchronization

Your Macros are synchronized in all your devices.

Security

Your data are safe and comunication with server are crypted.

Local executions

Your can execute Macros in your local device.

Distributed executions

Your can execute Macros in your local device and/or in other devices connected to your account.

Store

Your can download Macros created by other users from the store.

Social

Your can connect to others Sinapsi users, share your Macro with them, execute Macros in their devices (with their permission).

API

Developers can use the Sinaps's API to create new components and if they want publish them in the Sinapsi Store.

Brief story of Sinapsi

  • Computer Networks exam

    On the Occasion of exam of computer networks, we started to simplify our dreams regarding the super human AI for something doable. This was the birth of the Sinapsi project.

  • First commit

    April 2015

    In the beginning of April, we started to write the initial part of the Sinapsi Core and the Server code

  • First Alpha

    July 2015

    We completed the first pre-alpha 1.0 of Sinapsi Engine and the Sinapsi Android client. The Desktop client is still incomplete, the Sinapsi Android client is still bugged, but we are on track, and we hope that Sinapsi have success.
